El pasado 4 de Noviembre, Oracle anunció que "no se harán más lanzamientos de Oracle GlassFish Server con soporte comercial - específicamente Oracle GlassFish Server 4.x con soporte comercial de Java EE7 no será lanzado.". Si usted tiene aplicaciones en producción corriendo en GlassFish con soporte comercial, debería comenzar a explorar nuevas opciones.

Tenga en cuenta lo siguiente:
  • El código fuente cerrado implica una tecnología propietaria (vendor lock-in)
  • Como en la mayor parte de las migraciones, el código deberá ser reescrito.
  • La última lista de precios (17 de Octubre, 2013) * indica hasta un incremento de 4x en las tarifas de licencias y soporte a WebLogic Server Enterprise Edition.
Si usted está buscando una plataforma de aplicaciones de código abierto, con soporte comercial de un proveedor confiable, considere a Red Hat JBoss Enterprise Application Platform (EAP) y usted obtendrá:
  • Desarrollo rápido e incremento en la productividad con soporte para Java EE6 y otros framworks de desarrollo.
  • Una base privada para aplicaciones y servicios Java escalables, con built-in features para fallas, caching, balance de cargas inteligente e implementación distribuida.
  • Más que un bajo costo de entrada. Un reporte de IDC descubrió que luego de un período de 3 años, las compañías que utilizan EAP generaron un ROI (Return over investment) de 569% y recuperaron su inversión inicial en menos de 6 meses, luego de la implementación total.

Red Hat JBoss Enterprise Application Platform es la alternativa lógica a GlassFish. Sepa porqué.
